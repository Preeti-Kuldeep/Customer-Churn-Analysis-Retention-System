# Data Requirements Specification (DRS)

## Project Name

Customer Churn Analysis & Retention System

## Purpose

The purpose of this document is to define the data requirements for the Customer Churn Analysis & Retention System. This document serves as the foundation for database design, SQL development, data analysis, and reporting.

## Data Sources

The project will use the following data sources:

1. Customer Data
2. Subscription Data
3. Payment Data
4. Complaint Data

## Core Entities

The system consists of the following entities:

1. Customers
2. Subscriptions
3. Payments
4. Complaints
5. Customer Usage (Future Enhancement)

## Objectives of Data Collection

* Analyze customer churn patterns.
* Measure revenue loss.
* Identify high-risk customers.
* Analyze customer complaints.
* Support KPI calculations.
* Generate business reports and dashboards.

## Expected Dataset Size

| Table         | Estimated Records |
| ------------- | ----------------- |
| Customers     | 1,000             |
| Subscriptions | 1,000 - 1,500     |
| Payments      | 5,000 - 10,000    |
| Complaints    | 2,000 - 3,000     |
